Session: 2026-06-21 (later) — Feature #83 (Wave 1 — Platform)
"platform/config — loader & validation" — aggregated errors + unit tests
--------------------------------------------------------------------------------

WHAT WAS DONE
-------------
Promoted the previously-private cfg.validate() to an exported method
Config.Validate() and switched its error model from "fail on first issue"
to "aggregate every issue and return them all at once" using errors.Join
(stdlib, Go 1.20+). One boot attempt now reveals the full configuration
fault tree instead of one error at a time.

Code touched / produced this session:

  * apps/backend/internal/platform/config/config.go
      - Added three new fields per feature #83 spec:
          OTLPEndpoint   string  ← OTEL_EXPORTER_OTLP_ENDPOINT
          JWTSecretStub  string  ← JWT_SIGNING_SECRET
          EnableStubAuth bool    ← ENABLE_STUB_AUTH (default true in dev,
                                   false in staging/prod; spec calls for
                                   ENABLE_STUB_AUTH=false in production)
      - DBDSN() string method exposes DatabaseURL under the name the
        architecture spec uses (matches the feature description's
        "HTTPListenAddr, DBDSN, RedisURL, OTLPEndpoint, …" enumeration).
      - Load() collects every parse error from typed setters
        (getenvInt64/Int32/Bool/Duration) into a parseErrs slice and
        joins them with the result of Validate() before returning.
      - Validate() now appends to a []error slice and returns
        errors.Join(errs...). Required-field checks, range checks,
        enum checks, and cross-field rules (DEFAULT_LOCALE ∈
        ACTIVE_LOCALES, ENABLE_STUB_AUTH ↔ JWT_SIGNING_SECRET,
        ENABLE_STUB_AUTH must be false in production) all run regardless
        of earlier failures.
      - Two helpers added:
          looksLikePostgresDSN() — rejects mysql:// etc. early.
          redactDSN() — strips user:password@ from any DSN echoed back
            in error messages so a misconfigured prod URL can't leak
            credentials into a log line.

  * apps/backend/internal/platform/config/config_test.go (NEW)
      18 test functions covering:
        - Validate() OK path (valid baseline).
        - Empty struct → aggregated error mentions every required env var
          (APP_ENV, HTTP_LISTEN_ADDR, DATABASE_URL, DB_POOL_MAX_CONNS,
          BODY_LIMIT_BYTES, REQUEST_TIMEOUT_SECONDS, SHUTDOWN_TIMEOUT,
          DEFAULT_LOCALE, ACTIVE_LOCALES, LOG_FORMAT, LOG_LEVEL).
        - Invalid APP_ENV, LOG_LEVEL, LOG_FORMAT, DATABASE_URL scheme.
        - DSN credentials redacted from error message.
        - DB pool bounds (min > max, max = 0, min < 0).
        - BodyLimitBytes / RequestTimeout / ShutdownTimeout non-positive.
        - DEFAULT_LOCALE absent from ACTIVE_LOCALES.
        - Stub auth: requires secret when enabled; forbidden in prod;
          OK with empty secret when disabled.
        - DBDSN() == DatabaseURL alias.
        - IsProduction / IsDevelopment helpers.
        - Load(): missing required env vars → DATABASE_URL in error.
        - Load(): malformed BODY_LIMIT_BYTES → integer parse error in
          aggregated message.
        - Load() happy path with all env vars set.
      Tests use a small envSetter helper (record-and-restore on
      t.Cleanup) so each test starts from a clean env without polluting
      sibling tests or the parent shell.

  * .env.example
      Added ENABLE_STUB_AUTH=true above the existing JWT_SIGNING_SECRET
      block with the comment that production MUST set it to false.
